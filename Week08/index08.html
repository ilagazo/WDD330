Ch 8: CSS3 Transforms and Transitions

  • Transform: The CSS3 transform property lets you lets you translate, rotate, scale, and/or skew any element on the page.
  • Translation: Translation functions allow you to move elements left, right, up, or down.
  • Example: "transform: translate(45px, -45px);"
  • If you only want to move an element vertically or horizontally, you can use the translatex or translatey functions respectively.
  • Scaling: The scale(x,y) function scales an element by the defined factors horizontally then vertically. If only one value is provided, it will be used for both the x and y values, growing or shrinking your element or pseudo-element while maintaining the original aspect ratio. For example, scale(1) would leave the element the same size, scale(2) would double its proportions, scale(0.5) would halve them, and so on.
  • ScaleX or ScaleY lets you specifically scale dimensions. Remember that scale scales the element at it's origin. The origin, or element center never moves, just the size of the element does.
  • Rotation: The rotate() function rotates an element around the point of origin by a specified angle value. As with scale , by default the point of origin is the element’s center. Generally, angles are declared in degrees, with positive degrees moving clockwise and negative moving counterclockwise. In addition to degrees, values can be provided in grads, radians, or turns, but we’ll just be sticking with degrees.
  • Skew: The skew(x,y) function specifies a skew along the x and y axes. As you’d expect, the x specifies the skew on the x axis, and the y specifies the skew on the y axis. If the second parameter is omitted, the skew will only occur on the x axis.
  • There is also skewX and skewY as with the other functions.
  • We can also change the origin of the Transform using the "transform-origin" property.
  • REMEMBER: The order of transform functions does matter: if you rotate before translating, your translate direction will be on the rotated axis.
  • Transitions allow the values of CSS properties to change over time, essentially providing simple animations.
  • transition-property: The transition-property property defines the CSS properties of the element that should be transitioned, with all for all properties being the default.
  • transition-duration: The transition-duration property sets how long the transition will take: the duration of time it takes to go from the default state to the transitioned state. You can specify this either in seconds ( s ) or milliseconds ( ms ).
  • transition-timing-function: The transition-timing-function lets you control the pace of the transition in even more granular detail. Do you want your animation to start off slow and become faster, start off fast and end slower, advance at an even keel, or some other variation?
  • transition-delay: Finally, by using the transition-delay property, it’s possible to introduce a delay before the transition begins. Normally a transition begins immediately, so the default is 0 . Include the number of milliseconds ( ms ) or seconds ( s ) to delay the transition.
  • Shorthand transition: We can easily shorthand transition by including all of these just within the transition property. For example: "transition: transform 0.2s ease-out 50ms;"
  • Keyframes: @keyframes and @-webkit-keyframes are used to create animations (more like animation groups)
  • animation-name: This property is used to attach an animation (previously defined using the @keyframes syntax) to an element
  • animation-duration: The animation-duration property defines the length of time (in seconds or milliseconds) an animation takes to complete one iteration (all the way through, from 0% to 100%)
  • animation-timing-function: the animation-timing-function determines how the animation will progress over its duration.
  • animation-iteration-count: The animation-iteration-count property lets you define how many times the animation will play through.
  • animation-direction: Use this to reverse the keyframe (100% to 0%)
  • animation-delay: The animation-delay property is used to define how many milliseconds or seconds to wait before the browser begins the animation
  • Shorthand for animation: Just like with transform, we can shorthand animation all into the animation property. For example: "animation: 300ms ease-in alternate 5s backwards appearDisappear;"

Ch 12: Canvas, SVG, and Drag and Drop

  • Canvas: With HTML5’s Canvas API, we can draw anything we can imagine, all through JavaScript.
  • Create a canvas with the "canvas" tag. Note, The text in between the canvas tags will only be shown if the canvas element is not supported by the visitor’s browser.
  • With canvas the default width and height is 300 x 150.
  • Once a canvas element is stored in a js variable, we MUST setup the context. The context is the place where your drawing is rendered.
  • Once the context is rendered, we must fill our brush with color. This is done with the strokeStyle or fillStyle property.
  • Using fillReact and strokeReact methods, we are able to draw a rectangle on the canvas.
  • To draw a circle, we use the beginPath(), arc(), and closePath methods.
  • To save a Canvas drawing, meaning to store it locally, we can use the toDataUrl method. This method creates a URL with the image in it.
  • SVG stands for a scalable vector graphic. Instead of pixels, each "pixel" works as a vector using vector math. It scales and doesn't go blurry.
  • Drag and Drop API: This API allows us to specify that certain elements are draggable, and then specify what should happen when these draggable elements are dragged over or dropped onto other elements on the page.

Steps to create a simple transition using only CSS

  • 1. Declare the original state of the element in the default style declaration.
  • 2. Declare the final state of your transitioned element; for example, a :hover state.
  • 3. Include the transition functions in your default style declaration using the transition properties, including: transition-property , transition-duration , transition-timing-function , and transition-delay .
